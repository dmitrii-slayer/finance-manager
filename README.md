## Установка и запуск

### Требования:
- JDK 21+
- Maven 3.9+

1. Склонировать проект:
```bash
git clone git@github.com:dmitrii-slayer/finance-manager.git
cd finance-manager
```

2. Сборка проекта:
```bash
mvn clean package
```

3. Запуск приложения:
```bash
java -jar target/finance-manager-1.0-SNAPSHOT.jar
```

Или с использованием Maven:
```bash
mvn spring-boot:run
```

## Архитектура проекта

### Ключевые компоненты:

#### 1. Spec-First подход
- API сначала проектируется через OpenAPI спецификацию
- Автоматическая генерация интерфейсов и моделей

#### 2. Слоистая архитектура
- DTO: объекты передачи данных между слоями и внешними сервисами.
- Controller: Принимает HTTP запросы, валидирует входные данные
- Service: Содержит бизнес-логику, оркестрацию операций
- Repository: Обеспечивает доступ к данным через JPA
- Entities: Сущностные классы, представляющие таблицы базы данных.

### Основные сущности:

| Сущность | Описание |
|------|----------|
| User | Пользователь системы |
| Wallet | Кошелек пользователя |
| Category | Категория транзакций |
| Transaction | Финансовая транзакция |

### Бизнес-правила:
1. Каждый пользователь имеет ровно один кошелек
2. Категории могут быть типа INCOME (доходы) или EXPENSE (расходы)
3. Бюджеты устанавливаются только для категорий типа EXPENSE
4. Тип транзакции должен соответствовать типу категории
5. Баланс кошелька не может быть отрицательным
6. Бюджеты имеют периоды (MONTHLY, WEEKLY, YEARLY)
